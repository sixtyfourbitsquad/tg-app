Redgifs API Setup

✓
No account or credentials neededRedgifs provides a free temporary auth token automatically. Just set your search tags.
1

Set your search tags in .env

Comma-separated list of tags to fetch from Redgifs trending feed.

.env
REDGIFS_SEARCH_TAGS="nsfw,amateur,verified"
PIPELINE_LIMIT="25"
2

How it works

  • Pipeline fetches a temporary token from GET /v2/auth/temporary — no login required
  • Searches GET /v2/gifs/search?search_text=TAG&order=trending for each tag
  • Downloads the urls.hd MP4 directly — audio and video already combined, no ffmpeg needed
  • Uploads to GCS and saves to PostgreSQL with dedup check

Google Cloud Storage Setup

1

Create a GCP project

Go to console.cloud.google.com → New Project. Note your Project ID.

2

Create a Storage bucket

Cloud Storage → Buckets → Create

  • Name: e.g. nfws-videos-prod
  • Access control: Fine-grained
3

Create Service Account

IAM & Admin → Service Accounts → Create → Role: Storage Object Admin

4

Download JSON key and place it

bash
mkdir -p secrets
mv ~/Downloads/your-key.json secrets/gcp-service-account.json
5

Make bucket publicly readable

GCP Console → your bucket → Permissions → Grant Access → Principal: allUsers → Role: Storage Object Viewer

6

Update .env

.env
GCP_BUCKET="nfws-videos-prod"
GCP_PROJECT_ID="your-project-id-12345"
GCP_KEY_FILE="secrets/gcp-service-account.json"

Deploy to Ubuntu 22.04 VPS

✓
Automated optionThe deploy script handles everything in one command — Docker, Nginx, SSL, PM2 pipeline.

Option A — One-command automated deploy

bash — on your VPS
export DOMAIN="yourdomain.com"
export REPO_URL="https://github.com/sixtyfourbitsquad/tg-app.git"
sudo bash scripts/deploy.sh

Option B — Manual step by step

1. Install Docker

bash
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker

2. Clone repo and create .env

bash
git clone https://github.com/sixtyfourbitsquad/tg-app.git /var/www/nfws
cd /var/www/nfws
cp .env.example .env
nano .env  # fill in production values (use the credentials above)
mkdir -p secrets  # place gcp-service-account.json here

3. Start containers + run migrations

bash
docker compose up -d --build
docker compose exec app npx prisma migrate deploy
docker compose exec app npm run db:seed

4. Configure Nginx + SSL

bash
sudo apt install -y nginx certbot python3-certbot-nginx
sed "s/YOUR_DOMAIN/yourdomain.com/g" nginx.conf \
  | sudo tee /etc/nginx/sites-available/nfws
sudo ln -s /etc/nginx/sites-available/nfws /etc/nginx/sites-enabled/nfws
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d yourdomain.com

5. Start pipeline scheduler with PM2

bash
sudo npm install -g pm2 tsx
pm2 start tsx --name nfws-pipeline -- pipeline/scheduler.ts
pm2 save
pm2 startup   # copy + run the printed command

Monitor on VPS

bash
docker compose ps              # container status
docker compose logs -f app     # live app logs
pm2 logs nfws-pipeline         # pipeline logs
curl https://yourdomain.com/api/health | jq
