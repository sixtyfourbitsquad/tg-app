How the pieces fit

There is no scraping pipeline and no external media host. You pull videos from a Telegram channel to your PC, sync them to the VPS, and the Next.js app serves them from local disk through Nginx.

Step 1 · PC
Download

scripts/download-tg-channel.py pulls videos from a Telegram channel into ./downloaded_videos/.

Step 2 · PC → VPS
Deploy

scripts/deploy-videos.sh rsyncs new files to /home/adii/videos/ and runs the bulk uploader remotely.

Step 3 · VPS
Serve

Nginx streams /videos/*.mp4 directly from disk. Next.js returns feed JSON from Postgres.

ℹ
What lives where
  • Your PC: repo, Python + Telethon, downloaded .mp4 files, key.txt SSH key.
  • VPS: Docker (Postgres + Redis + Next app), Nginx, /home/adii/videos/, repo clone at ~/tg-app.
  • Browser: opens http://35.200.162.160 — mobile-shaped UI with Home / Trending / Saved / Profile.

Fill .env with exactly these keys:

.env
DATABASE_URL="postgresql://postgres:CHANGE_ME@postgres:5432/nfws_db?schema=public"
REDIS_URL="redis://redis:6379"

NEXTAUTH_SECRET="paste output of openssl rand -base64 32"

LOG_LEVEL="info"

VIDEOS_DIR="/home/adii/videos"

POSTGRES_PASSWORD="CHANGE_ME"   # must match DATABASE_URL
✕
Common mistakes
  • The password inside DATABASE_URL must match POSTGRES_PASSWORD exactly. Mismatch = permanent P1000 authentication failed.
  • VIDEOS_DIR must be an absolute path (no ~, no trailing slash).
  • Rotating NEXTAUTH_SECRET logs every user out. Keep a copy somewhere safe.

Configure Nginx

VPS

Nginx does two jobs: serve raw video files from /home/adii/videos/ at /videos/*, and proxy everything else to the Next app on :3000.

VPS bash
sudo nano /etc/nginx/sites-available/nfws

Paste this exactly:

nginx
server {
    listen 80 default_server;
    server_name _;

    client_max_body_size 50m;

    # Static videos — served directly from disk, bypasses Next.
    location /videos/ {
        alias /home/adii/videos/;
        add_header Accept-Ranges bytes;
        add_header Cache-Control "public, max-age=3600";
        access_log off;
        sendfile on;
        tcp_nopush on;
        types {
            video/mp4        mp4 m4v;
            video/webm       webm;
            video/quicktime  mov;
            video/x-matroska mkv;
        }
        default_type application/octet-stream;
    }

    # Everything else → Next.js
    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_set_header   Upgrade           $http_upgrade;
        proxy_set_header   Connection        "upgrade";
        proxy_read_timeout 60s;
    }
}

Enable and reload

VPS bash
sudo ln -sf /etc/nginx/sites-available/nfws /etc/nginx/sites-enabled/nfws
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t        # must print "syntax is ok" + "test is successful"
sudo systemctl reload nginx

Open the firewall

VPS bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw --force enable
⚠
Don't forget the GCP firewallIn the Google Cloud console: VPC network → Firewall → allow TCP:80 from 0.0.0.0/0. UFW alone is not enough on GCE instances.

API_ID      = "1234567"

ℹ
Channel identifiers
  • Public: "@channel_username" or "https://t.me/channel_username"

Download videos from the channel

PC
PC bash
python3 scripts/download-tg-channel.py

First-run login

  1. Enter your phone in international format: +919876543210.
  2. Telegram sends a login code to your Telegram app — type it in.
  3. If you have 2FA, enter your cloud password.

Telethon caches the session at scripts/tg_downloader.session. Future runs are silent.

What it does

  • Paginates newest → oldest in batches of BATCH_SIZE (500).
  • Keeps only messages with a video (message.video or document with video/* mime).
  • Skips anything larger than MAX_SIZE_MB (15 MB).
  • Skips files already on disk with a matching size, so re-runs are cheap.
  • Saves as <message_id>_<original_filename>.mp4 — unique per file, easy to trace back.
  • Uses an atomic .part → rename, so Ctrl-C is always safe.

Expected output

terminal
Batch 1: scanning 500 messages (offset_id=0)
  done  msg#4812 4812_clip.mp4
  skip (too large 22.1 MB) msg#4810 ...
  ...
──────── Summary ────────
Videos found        : 2451
Downloaded          : 1832
Skipped (exists)    : 519
Skipped (>15MB)     : 95
Skipped (non-video) : 4108
Errors              : 5
Output              : /home/meow/.../downloaded_videos

What happens under the hood

1

Pre-flight checks

Verifies key.txt permissions, counts .mp4 files, ensures /home/adii/videos/ exists on the VPS.

2

rsync upload

rsync -avh --partial --progress copies *.mp4 from PC to VPS. Resumable, skips files already present at the same size, prints live progress.

3

Remote DB registration

SSHes in and runs npx tsx scripts/bulk-upload.ts /home/adii/videos/. Each new .mp4 gets a Video row with url, file_path, file_size, and title. Already-registered filenames are skipped.

Troubleshooting

SymptomFix
P1000 authentication failed from PrismaPassword in DATABASE_URL doesn't match POSTGRES_PASSWORD. Edit .env, then docker compose down -v && docker compose up -d --build.
DATABASE_URL not found when running prisma on VPSRun it via docker compose exec app npx prisma ... — the env is loaded inside the container.
Migrations don't applydocker compose exec app npx prisma migrate deploy. Check docker compose logs postgres for errors.
npm install peer-dep errorAdd --legacy-peer-deps.
Video URL returns 404File isn't on the VPS. ssh ... 'ls /home/adii/videos | wc -l'. If empty, re-run deploy. If present, check Nginx error log: sudo tail -f /var/log/nginx/error.log.
Video URL returns 403Permissions. sudo chown -R adii:adii /home/adii/videos && sudo chmod -R 755 /home/adii/videos.
Nginx 502 Bad GatewayApp container isn't healthy. docker compose ps + docker compose logs -f app.
Deploy script: "SSH key not readable"chmod 600 key.txt. Verify path: ls -la key.txt.
Deploy script: "permission denied (publickey)"Public key not authorized on VPS. Add the key's .pub contents to ~adii/.ssh/authorized_keys on the VPS.
bulk-upload errors "Cannot find module @prisma/client"Ran on VPS host but deps aren't installed there. SSH in and: cd ~/tg-app && npm install --legacy-peer-deps && npx prisma generate.
Python: "could not resolve channel"You're not a member, or the id is wrong. Use -100... for private channels. Bots can't scrape history — use a user account.
Python: "FloodWaitError"Telegram rate-limited you. Stop, wait the indicated seconds (script prints it), then resume. Re-runs skip files already on disk.
First run asks for login code every timescripts/tg_downloader.session is getting deleted. Don't move it between machines.
Feed loads emptydocker compose exec postgres psql -U postgres -d nfws_db -c "SELECT COUNT(*) FROM videos;". If 0, the bulk-upload didn't insert. Run it manually: ssh ... "cd ~/tg-app && npx tsx scripts/bulk-upload.ts /home/adii/videos/".
Same video order on every reloadCheck /api/videos?sort=random — the seed field must differ each request. If not, client isn't generating a new seed. Hard-refresh the page.
Mute button has no effectiOS Safari blocks autoplay with sound; the initial tap unlock might not have registered. Tap anywhere on the video first, then the mute icon.
GCP VM unreachable on port 80VPC firewall rule missing. Console → VPC network → Firewall → Create: allow TCP 80 from 0.0.0.0/0.
Want to wipe DB and re-seeddocker compose exec postgres psql -U postgres -d nfws_db -c "TRUNCATE videos CASCADE;" then npx tsx scripts/bulk-upload.ts /home/adii/videos/.
